## 0. ひとことで言うと

時計・ハイブランド品・車の「いまの市場価値」を、**国内ECサイト（価格.com・楽天市場・Yahoo!ショッピング）の販売価格を集計**して査定する Web アプリ。査定した品物を**ポートフォリオ**（資産管理）や**欲しいものリスト**に登録できる。Netlify Functions 構成で、APIキー不要。

- ローカル確認：`node dev-server.js` → http://localhost:8888
- 同一Wi-Fi内のスマホから：http://192.168.0.141:8888 （PCのIP。サーバー起動中のみ）
- **まだ Netlify には本デプロイしていない**（ローカルのみ）

## 3. 実装済み機能

### 3-1. 検索カスケード（5ジャンル）
画面上から順に：

1. **⌚ 時計**：ブランド → モデル → 型番 → 文字盤（色）
   - 機械式ブランドを網羅（ロレックス〜スウォッチまで約50ブランド）
   - **型番は出品から自動抽出**し、現行モデル順・人気順で並べる
   - **文字盤色も出品名から自動学習**（「○○文字盤」表記を拾う。スレートローマ/ブラックサンダスト等の細分化も対応）
   - スウォッチのコラボ（ムーンスウォッチ/ブランパン/1965/ロイヤルポップ）対応
2. **💎 ハイブランド**：ブランド → カテゴリ → モデル →（サイズ/色/刻印）
   - クロムハーツ/エルメス/ヴァンクリーフ/カルティエ/ティファニー/ハリーウィンストン/シャネル/ルイヴィトン
   - **モデルは出品から自動学習**（クロムハーツ：スペーサー/ダガー/スクロール等、バンクリ：アルハンブラ系を区別）
   - **エルメス バーキン/ケリーはサイズ・色・刻印（製造年に変換）で絞込**可能
   - クロムハーツはサングラスも対応
3. **🚗 車**：メーカー → 車種
   - 国産（トヨタ/レクサス/日産/ホンダ/スズキ）+ 輸入（ポルシェ/フェラーリ/ベンツ/BMW等）
   - 「中古車」を必須ワードにしてパーツ・ミニカーを除外
   - ※年式絞り込みは一度実装したが、件数が減りすぎる・精度が落ちるため**ユーザー判断で撤去**
4. **🥇 金・プラチナ・銀（実物資産）**：種類 → 重量(g)
   - **田中貴金属の本日相場**（`/api/metals` = `netlify/functions/metals.js`）から買取グラム単価を取得
   - 評価額 = 買取単価 × 重量（純度の入力は不要にした＝純金/純プラチナ相当で計算）
   - 他ジャンルと違い**相場が一意なので時価が正確**。出品検索ではない
   - ポートフォリオの「時価を更新」では市場検索でなく最新グラム単価で再計算
5. **🔍 キーワード直接検索**：自由入力
   - モデル名だけ入力時はブランド名を自動補完（例：デイトナ→ロレックス デイトナ。バイク用品DAYTONA等の混入防止）

各ジャンルは**排他**（片方を選ぶと他がリセット）。

### 3-2. 査定結果
- 推定市場価値（中央値）、最安・中央・平均・最高、参照件数
- 外れ値除去：IQR上限 + 価格フロア（中央値の12%未満を除外）
- 出品一覧：**価格の高い順**で上位30件。各カードに画像・状態・付属品バッジ・**出品者名**
- 各出品カードに「☆ 欲しい」「＋ ポートフォリオ」ボタン

### 3-3. ポートフォリオ（資産管理）
- 写真（アップロード可・端末内で縮小保存）、品名、ブランド
- **購入価格**（クリック入力）、**推定市場価値**（自動 or 手動上書き）、**含み益**（市場価値−購入価格、％表示・緑赤）
- 価格更新日、購入日、状態
- 「⟳ 時価を更新」で市場価値を再取得（手動価格の銘柄は上書きしない）
- **ブランド別内訳** + **ジャンル別内訳**（時計/バッグ/リング/車 等）のバー表示
- ▲▼で並べ替え（スマホ向けにドラッグでなくボタン式）
- 「＋ 手動で追加」（検索せず品名+価格で登録）
- **売却済みは別テーブルに分離**：購入価格・推定市場価値・**売却価格（後から入力可）**・**損益**（売却価格−購入価格）
- サマリーに**含み益合計**・**実現損益合計**を表示

### 3-4. 欲しいものリスト
- ☆で登録。**登録時の時価**と**現在の時価**、**変動額（％）**を表示
- 「⟳ 時価を更新」で値動き追跡（買い時の判断用）
- 「購入→ポートフォリオへ」（確認ダイアログあり）で移動

## 4. 現状の課題（最重要・ここを次に検討）

### 🔴 課題1：本番デプロイがまだ（ローカルのみ）
- 現在は `node dev-server.js` のローカル起動のみ。**PCを閉じると使えない／外出先・スマホから常時アクセス不可**。
- コードは Netlify 用に書いてある（`netlify.toml` 設定済み）ので、**デプロイ作業をすれば解決**。
- ただしデプロイ前に下記の規約リスク（課題2）を要判断。

### 🔴 課題2：スクレイピングの規約・安定性リスク
- 価格.com・楽天・Yahoo! とも**公開APIではなくHTML解析**。各社の利用規約に抵触する可能性があり、**一般公開・商用提供は非推奨**（READMEにも明記）。個人利用・社内検証に留めるのが安全。
- **ページ構造が変わると壊れる**。実際この開発中に楽天の画像取得が一度全滅し、構造変更に追従して直した（`originalItemUrl`発見）。今後も各社のリニューアルで都度メンテが必要。
- アクセス集中でブロックされる可能性（レスポンスは5分キャッシュで緩和）。

### 🟡 課題3：商品リンク切れ（中古品の宿命）
- 中古は1点物のため、**表示後に売れるとリンク先が404/販売終了**になる。高い順表示で上位ほど希少→回転が速い。
- 価格.comの広告リダイレクトURL(`ksearch/jump/paid`)は直接URLに変換して改善済みだが、**「売り切れによるリンク切れ」自体は防げない**（再検索で最新在庫に入れ替わる）。
- 関連改善案（未実装）：**登録時に商品画像をこちら側で保存**しておけば、出品が消えてもポートフォリオの写真が残る。サーバーに画像中継エンドポイントを1本追加する必要あり。

### 🟡 課題4：車のデータが薄い
- 車の実車両出品は**実質Yahoo!ショッピングの中古車店のみ**。時計ほど件数が出ず、マイナー車種は0件になりやすい。
- 専門サイト（カーセンサー/グーネット）はJS描画で取得不可だった。

### 🟡 課題5：ジャンルごとの精度ムラ
- ハイブランドのバッグ等は付属品（保存袋・ハンドルカバー等）混入を都度除外パターンで潰している。**実例ベースで追記していく運用**が続く（文字盤色・モデル学習も同様）。
- まれに表記ゆれで似たモデル/色が分かれることがある。

### 🟢 課題6：データのブラウザ依存
- ポートフォリオ・欲しいものリストは **localStorage**（そのブラウザ内のみ）。
- スマホとPCでデータ共有されない、ブラウザのデータ消去で消える。
- 共有・永続化したい場合は、サーバー側ストレージ（要バックエンド）かエクスポート/インポート機能が必要。

## 5. 設計上の注意点（触る前に知っておくこと）

- **TLS設定**：`rejectUnauthorized: false`（TLS検証の無効化）は**禁止**（過去にauto-mode分類器に拒否された）。rolex-ringo用に中間証明書 `letsencrypt-yr2.pem` をバンドルしてCA補完する方式を採用済み（`fetchHtmlCompleteChain`）。
- **絞り込みの一貫性**：ブランド品のモデル/サイズ/色/刻印は「再検索」ではなく**同じ検索結果プールを正規表現で絞る**方式。これにより「件数表示と実際の結果が一致」する（CH+ と CHプラスの表記ゆれも吸収）。モール検索には深く取るためモデル語も付けるハイブリッド。
- **件数表示**：ドロップダウンの「(○件)」はモデル/サイズ/色/刻印では**外した**（選択後に取り直すと数が変わり紛らわしいため）。サイズ・色・刻印の連動件数は残している。
- **キャッシュ**：フロントは `fetch(..., { cache: "no-store" })` で常に最新取得。
- **rolex-ringo（ringo.js）**：コードは残すがUIから非表示（ユーザー指示）。
- **背景画像**：ヒーロー背景の写真表示は現在オフ（`renderHeroBackground()` をコメントアウト）。再開可能。設定は「左:クロムハーツCHクロス / 中央:白デイトナ / 右:バーキン エトゥープ」。
